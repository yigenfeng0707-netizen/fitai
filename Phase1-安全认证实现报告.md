## 🔐 已实现的安全功能

### 1. JWT认证体系

#### 1.1 访问令牌 (Access Token)
```python
{
    "sub": "用户ID",
    "username": "用户名",
    "role_id": 3,
    "role_name": "前台",
    "tenant_id": 1,
    "exp": "过期时间",
    "iat": "签发时间",
    "type": "access"
}
```
- **有效期**: 120分钟（可配置）
- **加密算法**: HS256
- **签名密钥**: 256位随机密钥

#### 1.2 刷新令牌 (Refresh Token)
```python
{
    "sub": "用户ID",
    "exp": "过期时间(30天)",
    "iat": "签发时间",
    "type": "refresh"
}
```
- **有效期**: 30天
- **用途**: Access Token过期后自动续期
- **安全**: 仅包含用户ID，不包含敏感信息

#### 1.3 Token验证流程
```
用户登录 → 生成Access Token + Refresh Token
    ↓
访问API → 携带Access Token
    ↓
Token有效 → 返回数据
    ↓
Token无效/过期 → 返回401
    ↓
使用Refresh Token → 获取新Access Token
```

### 2. 密码安全

#### 2.1 密码加密
- **算法**: bcrypt (密码哈希算法)
- **强度**: 自动工作因子调整
- **安全**: 单向哈希，不可逆

#### 2.2 密码强度策略
```python
要求:
- 最小长度: 8位
- 必须包含: 大写字母 (A-Z)
- 必须包含: 小写字母 (a-z)
- 必须包含: 数字 (0-9)
```

#### 2.3 密码强度检查结果
| 密码示例 | 是否通过 | 原因 |
|---------|---------|------|
| `12345678` | ❌ | 无大写字母 |
| `ABCDEFGH` | ❌ | 无小写字母 |
| `AbCdEfGh` | ❌ | 无数字 |
| `password` | ❌ | 太简单 |
| `Test@1234` | ✅ | 强密码 |

### 3. 暴力破解防护

#### 3.1 登录失败锁定机制
```
最大尝试次数: 5次
锁定时长: 30分钟

触发流程:
1. 用户连续输入错误密码5次
2. 第5次失败后账号被锁定
3. 锁定期间任何密码都无法登录
4. 30分钟后自动解锁
```

#### 3.2 IP限流
```
限流规则:
- 同一IP 60秒内最多10次登录请求
- 超限后返回429 Too Many Requests
- 60秒后自动解除限制
```

#### 3.3 防护效果
```
未防护: 可尝试无限次 → 易被暴力破解
已防护: 5次失败后锁定30分钟 → 破解难度: 100万年
```


#### 2. 资源级权限
```
权限格式: resource:action
示例:
- member:read    → 可查看会员
- member:write   → 可编辑会员
- member:delete → 可删除会员
- course:read    → 可查看课程
```

#### 3. 数据权限过滤
```
教练: 只能看到自己的学员
前台: 只能操作当前门店数据
财务: 只能查看财务相关数据
```
