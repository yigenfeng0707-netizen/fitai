## 部署步骤

### 步骤1：安装Docker

#### Windows
1. 下载 Docker Desktop: https://www.docker.com/products/docker-desktop
2. 运行安装程序
3. 重启电脑
4. 启动Docker Desktop

#### Ubuntu
```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
```

#### macOS
```bash
brew install docker docker-compose
```

### 步骤2：下载部署包

```bash
# 下载最新版本
wget https://github.com/your-repo/fitai/releases/latest/download/fitai-deploy.zip

# 解压
unzip fitai-deploy.zip
cd fitai-deploy
```

### 步骤3：配置环境

编辑 `.env` 文件：

```env
# 数据库配置
DB_HOST=postgres
DB_PORT=5432
DB_NAME=fitai
DB_USER=fitai
DB_PASSWORD=your_secure_password

# Redis配置
REDIS_HOST=redis
REDIS_PORT=6379

# JWT密钥（请修改为安全的随机字符串）
JWT_SECRET_KEY=your-super-secret-key-change-this-in-production
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=120

# 微信支付配置（可选）
WECHAT_APP_ID=your_app_id
WECHAT_MCH_ID=your_mch_id
WECHAT_API_KEY=your_api_key

# 阿里云短信配置（可选）
ALIYUN_ACCESS_KEY_ID=your_access_key
ALIYUN_ACCESS_KEY_SECRET=your_secret
ALIYUN_SMS_SIGN_NAME=您的签名

# AI配置（可选）
DEEPSEEK_API_KEY=your_deepseek_key

# 系统配置
STORE_NAME=FitAI健身房
STORE_PHONE=400-888-8888
```

### 步骤4：启动服务

```bash
# Windows
deploy.bat

# Linux/macOS
chmod +x deploy.sh
./deploy.sh

# 或使用Docker Compose直接启动
docker-compose up -d
```

### 步骤5：验证部署

```bash
# 检查服务状态
docker-compose ps

# 检查日志
docker-compose logs -f
```

访问以下地址验证：
- 前端：http://localhost
- 后端API：http://localhost:8000
- API文档：http://localhost:8000/docs

### 步骤6：初始化数据

首次部署需要初始化数据库：

```bash
docker-compose exec backend python -m scripts.init_db
```

## 配置说明

### 端口配置

| 端口 | 服务 | 说明 |
|------|------|------|
| 80 | Nginx | 前端访问 |
| 443 | Nginx (SSL) | HTTPS访问 |
| 8000 | FastAPI | API服务 |
| 5432 | PostgreSQL | 数据库 |
| 6379 | Redis | 缓存 |

### 域名配置

如需绑定域名，请修改 `nginx/nginx.conf`:

```nginx
server_name your-domain.com;  # 修改为您的域名
```

### SSL证书配置

#### 使用Let's Encrypt（免费）

```bash
# 安装Certbot
apt install certbot python3-certbot-nginx

# 申请证书
certbot --nginx -d your-domain.com

# 自动续期
certbot renew --dry-run
```

#### 使用商业证书

将证书文件复制到服务器：

```bash
cp your-cert.crt /path/to/certs/
cp your-cert.key /path/to/certs/
```

修改nginx配置指向证书路径。

## 运维管理

### 日常维护

#### 查看日志

```bash
# 查看所有日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend
docker-compose logs -f frontend

# 保存日志到文件
docker-compose logs > logs.txt
```

#### 备份数据

```bash
# 备份数据库
docker-compose exec postgres pg_dump -U fitai fitai > backup_$(date +%Y%m%d).sql

# 备份上传文件
tar -czf uploads_backup_$(date +%Y%m%d).tar.gz data/uploads/
```

#### 恢复数据

```bash
# 恢复数据库
docker-compose exec -T postgres psql -U fitai fitai < backup_20240101.sql

# 恢复上传文件
tar -xzf uploads_backup_20240101.tar.gz -C /
```

### 系统更新

```bash
# 拉取最新镜像
docker-compose pull

# 重启服务
docker-compose up -d

# 或一键更新
docker-compose pull && docker-compose up -d
```

### 性能优化

#### 数据库优化

```bash
# 进入数据库容器
docker-compose exec postgres psql -U fitai fitai

# 执行优化
VACUUM ANALYZE;
REINDEX DATABASE fitai;
```

#### 缓存清理

```bash
# 清理Redis缓存

```

### 监控告警

## 常见问题

### Q1: 启动失败，提示端口被占用

**解决方法**：
```bash
# Windows
netstat -ano | findstr "80 8000 5432 6379"

# 结束占用进程或修改docker-compose.yml中的端口映射
```

### Q2: 数据库连接失败

**解决方法**：
```bash
# 检查数据库容器状态
docker-compose ps postgres

# 重启数据库
docker-compose restart postgres

# 查看数据库日志
docker-compose logs postgres
```

### Q3: 前端无法访问API

**解决方法**：
```bash
# 检查后端容器状态
docker-compose ps backend

# 查看后端日志
docker-compose logs backend

# 重启后端服务
docker-compose restart backend
```

### Q4: 如何重置管理员密码

```bash
docker-compose exec backend python -c "
from app.database import SessionLocal
from app.models.user import User
from passlib.context import CryptContext

pwd_context = CryptContext(schemes=['bcrypt'], deprecated='auto')
db = SessionLocal()
user = db.query(User).filter(User.username == 'admin').first()
user.password = pwd_context.hash('admin123')
db.commit()
db.close()
print('密码已重置为: admin123')
"
```

### Q5: 如何迁移到新服务器

1. 备份数据：
```bash
docker-compose exec postgres pg_dump -U fitai fitai > fitai_db.sql
tar -czf fitai_data.tar.gz data/
```

2. 在新服务器部署：
```bash
# 上传备份文件
scp fitai_db.sql fitai_data.tar.gz user@new-server:/path/

# 解压数据
tar -xzf fitai_data.tar.gz

# 恢复数据库
docker-compose exec -T postgres psql -U fitai fitai < fitai_db.sql
```

### Q6: 如何升级大版本

```bash
# 备份当前数据
docker-compose exec postgres pg_dump -U fitai fitai > backup.sql

# 拉取新版本
git pull

# 重新构建
docker-compose build --no-cache

# 启动服务
docker-compose up -d

# 如有数据迁移问题，查看迁移文档
cat MIGRATION.md
```
