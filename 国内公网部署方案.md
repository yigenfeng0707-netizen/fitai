## 一、部署架构概览

### 推荐方案：阿里云/腾讯云 + Docker Compose

```
│                     (手机/电脑/平板)                            │
└─────────────────────────────────────────────────────────────────┘
                              │

┌─────────────────────────────────────────────────────────────────┐
│                    阿里云/腾讯云服务器                            │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │                   Nginx (反向代理)                       │    │

│                              │                                 │

│  ┌─────────────────┐ ┌─────────────┐ ┌─────────────────┐      │
│  │  前端静态文件    │ │  API服务    │ │  WebSocket     │      │
│  │  (80/443)       │ │ (8000)      │ │  (8001)        │      │
│  └─────────────────┘ └─────────────┘ └─────────────────┘      │

│  └───────────────┘ └───────────────┘ └───────────────┘        │
└─────────────────────────────────────────────────────────────────┘
```

## 二、服务器要求

## 四、详细部署步骤

### 第一步：服务器初始化

```bash
# 连接服务器
ssh root@你的服务器IP

# 更新系统
apt update && apt upgrade -y

# 安装必要工具
apt install -y curl wget git vim unzip

# 安装Docker
curl -fsSL https://get.docker.com | sh
systemctl enable docker
systemctl start docker

# 安装Docker Compose
curl -L "https://github.com/docker/compose/releases/download/v2.20.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose

# 验证安装
docker --version
docker-compose --version
```

### 第二步：配置防火墙

```bash
# 开放必要端口
ufw allow 22    # SSH
ufw allow 80    # HTTP
ufw allow 443   # HTTPS
ufw allow 8000  # API（可选，如果不用nginx）
ufw enable
```

### 第三步：创建项目目录

```bash
mkdir -p /www/fitai
cd /www/fitai

# 创建必要目录
mkdir -p ssl nginx/data logs postgres redis backups
```

### 第四步：配置SSL证书（免费）

使用Let's Encrypt免费证书：

```bash
# 安装certbot
apt install -y certbot python3-certbot-nginx

# 申请证书（需要域名已解析）
certbot certonly --nginx -d yourdomain.com -d www.yourdomain.com

# 证书会自动保存到：
# /etc/letsencrypt/live/yourdomain.com/
```

### 第五步：配置Nginx

创建 `/www/fitai/nginx/nginx.conf`：

```nginx
upstream api_backend {
    server 127.0.0.1:8000;
}

server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name yourdomain.com www.yourdomain.com;

    # SSL配置
    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;

    # 前端静态文件
    root /www/fitai/nginx/data;
    index index.html;

    # Gzip压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;

    # 前端路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # API代理
    location /api/ {
        proxy_pass http://api_backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时配置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # WebSocket代理
    location /ws/ {
        proxy_pass http://api_backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 日志
    access_log /www/fitai/logs/nginx_access.log;
    error_log /www/fitai/logs/nginx_error.log;
}

# API子域名（可选）
server {
    listen 443 ssl http2;
    server_name api.yourdomain.com;

    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

### 第七步：配置Docker Compose

创建 `/www/fitai/docker-compose.yml`：

```yaml
version: '3.8'

services:
  # Nginx反向代理
  nginx:
    image: nginx:alpine
    container_name: fitai-nginx
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/data:/www/fitai/nginx/data:ro
      - ./logs:/www/fitai/logs
      - /etc/letsencrypt:/etc/letsencrypt:ro
    depends_on:
      - backend
    restart: unless-stopped
    networks:
      - fitai-network

  # 后端API服务
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: fitai-backend
    env_file:
      - .env
    volumes:
      - ./backups:/www/fitai/backups
    depends_on:
      - postgres
      - redis
    restart: unless-stopped
    networks:
      - fitai-network
    command: >
      sh -c "alembic upgrade head && 
             uvicorn app.main:app --host 0.0.0.0 --port 8000"

  # PostgreSQL数据库
  postgres:
    image: postgres:15-alpine
    container_name: fitai-postgres
    environment:
      POSTGRES_USER: fitai_user
      POSTGRES_PASSWORD: your-strong-db-password
      POSTGRES_DB: fitai_db
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    restart: unless-stopped
    networks:
      - fitai-network
    command: >
      postgres 
      -c max_connections=200
      -c shared_buffers=256MB
      -c effective_cache_size=512MB
      -c maintenance_work_mem=64MB
      -c checkpoint_completion_target=0.9
      -c wal_buffers=16MB
      -c default_statistics_target=100
      -c random_page_cost=1.1
      -c effective_io_concurrency=200
      -c work_mem=4MB
      -c min_wal_size=1GB
      -c max_wal_size=4GB

  # Redis缓存
  redis:
    image: redis:7-alpine
    container_name: fitai-redis
    command: redis-server --requirepass your-redis-password --maxmemory 512mb --maxmemory-policy allkeys-lru
    volumes:
      - redis_data:/data
    ports:
      - "6379:6379"
    restart: unless-stopped
    networks:
      - fitai-network

  # 定时备份任务
  backup:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: fitai-backup
    env_file:
      - .env
    volumes:
      - ./backups:/www/fitai/backups
      - postgres_data:/var/lib/postgresql/data:ro
    depends_on:
      - postgres
    restart: unless-stopped
    networks:
      - fitai-network
    command: >
      sh -c "while true; do 
               sleep 86400; 
               /app/scripts/backup.sh;
             done"
    # 每天凌晨2点执行备份

networks:
  fitai-network:
    driver: bridge

volumes:
  postgres_data:
  redis_data:
```

### 第八步：创建备份脚本

创建 `/www/fitai/backend/scripts/backup.sh`：

```bash
#!/bin/bash
set -e

# 配置
BACKUP_DIR="/www/fitai/backups"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_NAME="fitai_backup_${DATE}.sql"
KEEP_DAYS=30

# 创建备份
echo "开始备份数据库..."
PGPASSWORD=your-strong-db-password pg_dump -h postgres -U fitai_user -d fitai_db -F c -f "${BACKUP_DIR}/${BACKUP_NAME}"

# 压缩备份
gzip "${BACKUP_DIR}/${BACKUP_NAME}"

# 清理旧备份
find ${BACKUP_DIR} -name "fitai_backup_*.sql.gz" -mtime +${KEEP_DAYS} -delete

echo "备份完成: ${BACKUP_NAME}.gz"
```

## 六、安全加固

### 1. 服务器安全

```bash
# 创建新用户
adduser deploy
usermod -aG sudo deploy

# 配置SSH密钥登录
mkdir -p ~/.ssh
chmod 700 ~/.ssh
cat >> ~/.ssh/authorized_keys << EOF
your-public-key-here
EOF
chmod 600 ~/.ssh/authorized_keys

# 禁用密码登录和root登录
sed -i 's/#PermitRootLogin yes/PermitRootLogin no/' /etc/ssh/sshd_config
sed -i 's/#PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
systemctl restart sshd
```

### 2. 配置防火墙规则

```bash
# 只开放必要端口
ufw default deny incoming
ufw allow ssh
ufw allow http
ufw allow https
ufw enable
```

### 3. 配置_fail2ban_防暴力破解

```bash
apt install -y fail2ban
systemctl enable fail2ban

cat > /etc/fail2ban/jail.local << EOF
[nginx-http-auth]
enabled = true
port = http,https
filter = nginx-http-auth
logpath = /www/fitai/logs/nginx_error.log
maxretry = 5
bantime = 3600
findtime = 600

[nginx-noscript]
enabled = true
port = http,https
filter = nginx-noscript
logpath = /www/fitai/logs/nginx_access.log
maxretry = 10
bantime = 7200
findtime = 600
EOF

systemctl restart fail2ban
```

## 八、售后服务与监控

### 1. 配置监控告警

```bash
# 安装监控脚本
cat > /www/fitai/monitor.sh << 'EOF'
#!/bin/bash

# 检查服务状态
if ! docker-compose ps | grep -q "Up"; then
    echo "服务异常！" | mail -s "FitAI服务告警" admin@yourdomain.com
    # 发送钉钉通知
    curl -H "Content-Type: application/json" \
         -d "{\"msgtype\":\"text\",\"text\":{\"content\":\"FitAI服务异常\"}}" \
         https://oapi.dingtalk.com/robot/send?access_token=YOUR_TOKEN
fi

# 检查磁盘空间
DISK_USAGE=$(df -h / | tail -1 | awk '{print $5}' | sed 's/%//')
if [ $DISK_USAGE -gt 80 ]; then
    echo "磁盘空间不足！" | mail -s "FitAI磁盘告警" admin@yourdomain.com
fi
EOF

# 添加定时任务
echo "*/5 * * * * /www/fitai/monitor.sh" >> /var/spool/cron/crontabs/root
```
